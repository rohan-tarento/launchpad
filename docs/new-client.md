# New programme onboarding (v0.5.10)

Quick checklist. Full walkthrough: **[greenfield.md](greenfield.md)**.

---

## Checklist

### Day 0 — Local setup

- [ ] Install Launchpad: `pipx install "launchpad @ git+https://github.com/drivestream-lab/launchpad@v0.5.10"`
- [ ] Run `launchpad onboard interview` (4 questions → 5 YAML files + registry + env stub)
- [ ] Set `GITHUB_TOKEN` in `~/.config/launchpad/env.d/<slug>.env`
- [ ] `chmod 600 ~/.config/launchpad/env.d/<slug>.env`
- [ ] `launchpad --client <slug> doctor`

### Day 1 — Meta repo on GitHub

- [ ] `launchpad init-client --meta --dry-run` (preview)
- [ ] `launchpad init-client --meta --apply` (creates team, repo, gitflow, board)
- [ ] Clone meta: `gh repo clone <org>/<slug>-meta`

### Day 1 — Scaffold meta (optional)

- [ ] Edit `config/scaffold-<org>.yaml`: set `meta.enabled: true`, fill `template` + `ref` + `context`
- [ ] `launchpad apply-scaffold --meta --apply`

### Day 1 — Pin meta harness

- [ ] Edit `config/harness-<org>.yaml`: set `constitution.ref` to a pinned tag
- [ ] `launchpad apply-harness --meta --apply`
- [ ] `launchpad check-harness --meta`

### Day N — Add each app repo

- [ ] Edit `config/governance-<org>.yaml`: add repo block (stack + teams required)
- [ ] `launchpad init-client --repo <name> --apply`
- [ ] Edit `config/scaffold-<org>.yaml`: add repo block if scaffolding
- [ ] `launchpad apply-scaffold --repo <name> --apply`
- [ ] `launchpad apply-harness --repo <name> --apply`
- [ ] `launchpad check-harness --repo <name>`
- [ ] Edit `config/service-catalog-<org>.yaml`: promote repo from `planned` → `live`

---

## Key principles

- **YAML is SSOT** — no CLI flags override config
- **All commands are dry-run by default** — add `--apply` to execute
- **All commands are idempotent** — safe to re-run after fixing config
- **`--meta` or `--repo <name>` required** — no `--all` flag
